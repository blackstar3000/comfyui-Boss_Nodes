# Character Builder CRUD + Preview Images — Design Spec

**Date:** 2026-07-20
**Status:** Approved
**Scope:** Add full CRUD (Add/Edit/Delete) and character preview images to the Ultimate Character Builder Pro node.

---

## 1. Overview

The Ultimate Character Builder Pro node currently has **zero CRUD** — characters, expressions, and poses are read-only JSON files. This spec adds:

- Per-item save/delete backend endpoints (matching Artist Selector pattern)
- CRUD UI in the editor (Add/Edit/Delete, sub-modals, toasts, refresh)
- Character preview images from nax.moe (`danbooru-character-tags-v4.5`)
- Custom preview URL support (paste any URL)
- Backend image proxy to bypass CDN hotlinking

---

## 2. Backend Changes

### 2.1 New Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `POST /char_boss/save` | POST | Add or update a character/expression/pose entry |
| `POST /char_boss/delete` | POST | Remove an entry by name |
| `GET /char_boss/proxy_image` | GET | Proxy external image URLs (CDN bypass) |

### 2.2 Save Endpoint — `POST /char_boss/save`

**Request body:**
```json
{
  "type": "characters",
  "name": "ganyu_(genshin impact)",
  "prompt": "ganyu (genshin impact), genshin impact, 1girl, ...",
  "categories": ["Female"],
  "custom_preview": "https://cdn.donmai.us/..."
}
```

- `type` is one of: `"characters"`, `"expressions"`, `"poses"`
- `name` is the entry key (required)
- `prompt` is the danbooru-style prompt text (required)
- `categories` is a list of category names (can be empty)
- `custom_preview` is an optional URL for a custom preview image (can be empty/null)

**Logic:**
1. Load the appropriate Collection (`_CHARACTERS`, `_EXPRESSIONS`, or `_POSES`)
2. Check if `name` already exists in `coll.items`
3. If new: create entry as `{ "prompt": prompt, "custom_preview": custom_preview }`
4. If existing: update prompt; preserve existing `custom_preview` if new one not provided
5. Save to disk via `coll.save()`
6. Return `{ "name", "prompt", "categories", "custom_preview", "is_new": true/false }`

**Backward compatibility:** Old entries are stored as plain strings (`"key": "prompt text"`). New entries use object format (`"key": { "prompt": "...", "custom_preview": "..." }`). The Collection loader already handles both formats — the `Collection` class from `cache_utils.py` reads JSON as-is. The JS frontend reads `typeof entry === "string" ? entry : entry.prompt`.

### 2.3 Delete Endpoint — `POST /char_boss/delete`

**Request body:**
```json
{
  "type": "characters",
  "name": "ganyu_(genshin impact)"
}
```

**Logic:**
1. Load the appropriate Collection
2. Remove `name` from `coll.items`
3. Remove `name` from `coll.categories` (all category lists)
4. Save to disk
5. Return `{ "name", "deleted": true }`

### 2.4 Proxy Endpoint — `GET /char_boss/proxy_image`

Same pattern as `/wai_artist/proxy_image`:
- Accepts `?url=<encoded_url>` query parameter
- Fetches the URL server-side with proper `User-Agent` and `Referer` headers
- Returns the image bytes with `Content-Type` header
- Used for nax.moe CDN URLs and custom preview URLs that block hotlinking

### 2.5 Refresh Endpoint — `POST /char_boss/refresh`

Already exists. Force-reloads all three JSON files from disk.

### 2.6 Data Endpoint — `GET /char_boss/data`

Already exists. Returns flattened libraries + categories. **Will be extended** to also return:
- `character_previews`: `{ "character_name": "https://cdn.zele.st/..." }` (from `character_previews.json`)

## 3. Data Files

### 3.1 `py/character_previews.json` (NEW)

**Format:**
```json
{
  "ganyu_(genshin impact)": "https://cdn.zele.st/data/NAX/Images/danbooru-character-tags-v4.5/ganyu_(genshin impact).jpg",
  "hu_tao_(genshin impact)": "https://cdn.zele.st/data/NAX/Images/danbooru-character-tags-v4.5/hu_tao_(genshin impact).jpg"
}
```

Maps character names to CDN preview URLs. Generated by `fetch_character_previews.py`.

### 3.2 `py/fetch_character_previews.py` (NEW)

**Purpose:** Scrape `https://nax.moe/?gallery=danbooru-character-tags-v4.5` to build `character_previews.json`.

**Pattern:** Same as `py/fetch_artist_previews.py`:
1. Fetch the nax.moe gallery page(s)
2. Extract character tag names and their associated image URLs
3. Output to `py/character_previews.json`

### 3.3 Entry Format Migration

Current format (string):
```json
{ "characters": { "ganyu_(genshin impact)": "ganyu (genshin impact), genshin impact, ..." } }
```

New format (object, backward-compatible):
```json
{ "characters": { "ganyu_(genshin impact)": { "prompt": "ganyu (genshin impact), genshin impact, ...", "custom_preview": "" } } }
```

**Important:** Existing string entries continue to work unchanged. Only newly created/edited entries use the object format. The JS frontend handles both:
```js
const prompt = typeof entry === "string" ? entry : entry.prompt;
const customPreview = typeof entry === "object" ? (entry.custom_preview || "") : "";
```

## 4. Frontend Changes

### 4.1 Editor Layout

The Character Builder editor already has a left panel with three sections (Character, Expression, Pose). Each section will gain:

- **+ Add** button — opens sub-modal for new entry
- **Refresh** button — re-fetches library data
- **Edit icon** (pencil) — hover-reveal on each list item
- **Delete icon** (X) — hover-reveal on each list item, confirmation toast

### 4.2 Sub-Modal (Add/Edit)

**Fields:**
- **Name** (text input) — disabled on edit
- **Prompt** (textarea) — the danbooru-style prompt text
- **Categories** (toggle chips) — highlighted = selected, click to toggle on/off
- **Preview Image URL** (text input) — paste a URL, stored as `custom_preview`

**Buttons:**
- **Save** — POSTs to `/char_boss/save`, re-fetches data, refreshes list, shows toast
- **Cancel** — closes modal

**Toast messages:**
- Add: `"ganyu_(genshin impact) added"`
- Edit with category changes: `"ganyu_(genshin impact): added to: Female; removed from: Male"`
- Edit without category changes: `"ganyu_(genshin impact) updated"`
- Delete: confirmation toast `"Delete ganyu_(genshin impact)?"` with Yes/No

### 4.3 Preview Thumbnails

Each list item shows a 48x48 thumbnail when available:

**Source priority:**
1. `custom_preview` from the entry object (if set)
2. `character_previews.json` URL (from nax.moe scrape)

**Proxy:** All external URLs are loaded through `/char_boss/proxy_image?url=<encoded>` to bypass CORS and hotlink blocking.

**Fallback:** If no preview available, show a placeholder icon (same as Artist Selector).

### 4.4 CSS

Reuse existing shared classes from `boss_theme`:
- `boss-widget`, `boss-widget-head`, `boss-btn-open`, `boss-status`
- `boss-art-*` classes for list items, thumbnails, meta, category chips
- `boss-art-submodal-overlay`, `boss-art-submodal` for sub-modals
- `boss-art-crud-btn` for CRUD buttons

No new CSS needed — all patterns already exist in the Artist Selector.

## 5. Data Flow

```
1. Editor opens → GET /char_boss/data → loads characters, expressions, poses + previews
2. User clicks + Add → sub-modal opens with empty fields
3. User fills fields → Save → POST /char_boss/save → re-fetch data → refresh list
4. User hovers list item → pencil + X icons appear
5. Click pencil → edit sub-modal pre-filled → Save → same flow
6. Click X → confirmation toast → Yes → POST /char_boss/delete → re-fetch → refresh
7. Refresh button → POST /char_boss/refresh → re-fetch → refresh
```

## 7. Constraints

- **100% backward compatibility** — node names, categories, inputs, outputs, widget names unchanged
- **Existing string entries work** — no forced migration, both string and object formats supported
- **No new dependencies** — uses existing `aiohttp` for proxy, existing `Collection` class for persistence
- **Follow established patterns** — Artist Selector CRUD + Scene Maker Pro save patterns
